本项目实现了一个检索增强生成（RAG）管道，允许用户针对自定义文档库进行自然语言提问。后端使用 Python 和 LangChain 构建，负责文档摄取、向量嵌入以及基于大语言模型的答案生成。React 前端提供了简洁的聊天界面，用户可以上传文档、提出问题，并在界面中直接查看来源引用。

文档摄取管道会将上传的 PDF 和 Markdown 文件拆分为语义完整的片段，使用 OpenAI 的嵌入模型生成向量，并将其存储在本地 FAISS 索引中。查询时，系统会检索最相关的片段，将其作为上下文输入语言模型，然后返回带有原始文档引用的答案。

项目的主要挑战包括：管理文本切分边界以避免在句子中间截断、调整检索阈值以平衡精确率和召回率，以及逐 token 流式传输长篇答案以保持界面响应速度。项目以 Docker Compose 方式部署，包含 API 服务器、向量存储和 Nginx 反向代理三个独立容器。
