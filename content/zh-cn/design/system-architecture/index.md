本项目汇集了我在过去一年中为各类软件系统创建的架构图。每张图都传达了系统的高层结构——服务组成、数据流向、外部集成和部署拓扑——让工程师和相关方都能一目了然。大部分图使用 Draw.io 绘制以确保技术精确性，在需要出现在演示文稿或文档站点中时，再用 Figma 进行润色。

图集包括展示服务边界和 API 契约的微服务分解图、展示消息代理和消费者组的事件驱动架构图，以及描绘 VPC、负载均衡器和容器编排集群的云基础设施图。每张图都遵循统一的视觉语言：按服务层级使用不同颜色编码、为数据库、队列和缓存使用标准化图标，并用清晰的方向箭头区分同步和异步通信。

制作有效的架构图，关键在于取舍。我力求让每张图只聚焦于一个关注点——部署、数据流或领域边界——避免让读者被过多细节所淹没。分层视图使读者可以从万米高空的全局概览逐步深入到单个服务的内部细节。
